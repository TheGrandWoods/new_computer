A/An Plugin

When using some plugins, like the tap-plugin, you can run into an annoying problem. Here's an example: 

tap = \{import:tap-plugin\}

output
  That's \{a\} \[tap(animal)\]

animal
  antelope
  zebra
  ...

The tap button only randomizes the \[tap(animal)\] part when you click it, and leaves all the other parts the same. This is a problem because the \{a\} part doesn't get updated to match the new animal. So the initial sentence that was generated might have been "That's an antelope" and then you tap antelope and it changes to "zebra" so now the sentence reads "That's an zebra" which is incorrect. The \{a\} part doesn't update because the whole purpose of the tap plugin is to only update the part that you tapped.

So this plugin is a replacement for \{a\} which also works in situations like the one explained above. Here's how to use it: 

an = \{import:a-an-plugin\}
tap = \{import:tap-plugin\}

output
  That's \[an()\] \[tap(animal)\].

You can use \[an().titleCase\] to get A/An and \[an().upperCase\] to get A/AN.
