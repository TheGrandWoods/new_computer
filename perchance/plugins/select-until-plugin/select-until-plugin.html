selectUntil Plugin

This plugin allows you to keep trying to select an item from a given list until the resulting item meets your specified requirements.

Below is a simple demonstration of how to use it. In this example we're trying to select a prefix and a suffix to construct a name, but we don't want the prefix and the suffix to be the same. 

selectUntil = \{import:select-until-plugin\}

prefix
  feather
  fire
  // ...

suffix
  feather
  foot
  // ...

output
  Her name was \[p = prefix.selectOne\]\[s = selectUntil(suffix, item=> item.evaluateItem != p.evaluateItem)\].

You can change the item.evaluateItem != p.evaluateItem part to any condition you like (read about conditions in the Dynamic Odds and If/Else sections of perchance.org/examples).

Note that we need to write item.evaluateItem != p.evaluateItem rather than just item != p. That's because evaluateItem converts the item into "raw" text, and then we check whether the text matches. If we write item != p, then we're asking whether item is "literally the exact same list item as p", which would never be true - because item and p are two different items from two different lists. In other words, we don't want to compare the "identities" of the list items, we want to see whether they have the same text.

As a slightly more complex example, if you wanted the suffix to not be the same as the prefix, but also to have a sub-property called "type" that's equal to "air", you'd write this: 

...

suffix
  feather
    type = air
  stone
    type = earth

output
  Her name was \[p = prefix.selectOne\]\[s = selectUntil(suffix, item=> item.evaluateItem != p.evaluateItem && item.type == "air")\].

Your requirement/condition always needs to start with the item=> thing that you see in the above examples. It just tells the Perchance engine that you're writing a condition and that item is the name of the "placeholder" that you're using to represent the selected item that's being tested.

Notes:

  • Here's an example of how to use this plugin.
  • You can actually change item=> to blah=> or any other name you want - that just means that you have to write blah instead of item in your condition code.
  • It's possible to write a condition that cannot possibly be satisfied. In that case this plugin will give up after 10000 attempts and display an error.
  • Check out more plugins at perchance.org/plugins
